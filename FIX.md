# FIX.md

## 目的

このファイルは、work-2 リポジトリにおける修正・改善・不具合対応の標準ルールである。

思いつきの修正、場当たり的な変更、設定ブレ、デザイン崩れ、混同リスクの増加を防ぐため、すべての修正作業はこのファイルに従う。

---

## 修正の基本姿勢

修正は、ただ壊れている箇所を直す作業ではない。
作品の体験、物語、導線、安全性、デザイン品質を守る作業である。

修正時に守ること。

* 変更目的を明確にする
* 変更範囲を絞る
* 関係ない箇所を触らない
* PC表示を基準に確認する
* スマホ表示も必ず確認する
* 固有名詞・日付・文体を勝手に変えない
* フィクションヘッダーを削除しない
* ページ固有の footer を雑に共通化しない
* 混同リスクを増やさない
* 参考デザインのコピーをしない
* 変更後にリンクと導線を確認する

---

## 修正前に確認する資料

修正を始める前に、以下を確認する。

1. `work-2-設定.txt`
2. `ARG作品制作における不変原則集`
3. `AGENTS.md`
4. `DESIN.md`
5. 変更対象ファイル

7. 既存CSS

9. 現在のルーティング / URL構造

## 修正の優先順位

修正は、以下の優先度で扱う。

### P0：即修正

公開・体験・安全性に重大な問題があるもの。

例。

* 実在機関と誤認される
* 実在の電話番号・住所・通報先が出ている
* フィクション表記がない
* 重要ページが開けない
* 最終行動ができない
* フォームが実在機関に送信されるように見える
* PCで完全に崩れている
* スマホで本文が読めない
* 主要導線が切れている
* ネタバレページに意図せず直アクセスできる

### P1：優先修正

作品体験を大きく損なうもの。

例。

* 固有名詞が設定資料と違う
* 日付が矛盾している
* 人物の文体が違う
* ページの役割が不明
* 検索結果が別タブで開かない
* ヘッダーが二重に出ている
* フッターが共通テンプレートのまま
* PC基準のデザインになっていない
* 参考デザインに似すぎている
* スマホでボタンが押しづらい

### P2：通常修正

品質向上のために行うもの。

例。

* 余白調整
* 文字サイズ調整
* ボタン文言改善
* カードUI改善
* 記事本文の自然化
* 画像提案
* 説明文の強化
* 導線の見やすさ改善

### P3：後回し可能

公開後でも対応できる軽微な改善。

例。

* 細かいアニメーション
* 装飾追加
* 軽微な表記ゆれ
* 内部コメント整理
* CSSの整理

### 1. 現状確認

* 関連リンクを確認する

## PC基準の修正ルール

デザイン崩れ修正では、必ずPCを基準にする。

PCで確認すること。

* 余白が詰まりすぎていないか
* 文字が小さすぎないか
* ファーストビューが安っぽくないか
* 情報の階層が分かるか
* 画像と本文のバランスがよいか
* ページの媒体感が出ているか
* 読み物として疲れないか
* 導線が自然か

スマホで確認すること。

* 横スクロールがないか
* 文字が読めるか
* 画像がはみ出していないか
* ボタンが押せるか
* ヘッダーが邪魔していないか
* フッターが大きすぎないか
* 重要情報が消えていないか

---

## ヘッダー修正ルール

フィクションヘッダーは削除禁止。
変更する場合は、ページ種別に合わせた表記変更のみ可。

許可例。

* FICTIONAL STORY INTERFACE
* FICTIONAL SEARCH INTERFACE
* FICTIONAL ARCHIVE INTERFACE
* FICTIONAL NEWS ARCHIVE
* FICTIONAL DIARY INTERFACE
* FICTIONAL LETTER ARCHIVE

禁止。

* ヘッダー削除
* ヘッダーを display:none にする
* ヘッダーを本文に埋もれさせる
* 実在機関風の公式ヘッダーに置き換える
* ページによってフィクション表記を消す

ヘッダーが二重になっている場合は、どちらか一方に統一する。
ただし、完全削除ではなく、1つだけ残す。

---

## フッター修正ルール

共通フッターを全ページで同じ見た目にしない。
ページの世界観に合わせた fictional footer にする。

修正時に確認すること。

* フッター文言がページ内容に合っているか
* AI画像の注記が必要か
* 実在機関との関係否定が必要か
* フッターが大きすぎないか
* フッターが小さすぎて読めないか
* PCとスマホで破綻していないか

基本文言例。

このページは創作作品内の架空Webページです。
実在の人物・団体・施設とは関係ありません。
本作品にはAI生成画像または加工画像を含む場合があります。

## 検索ページ修正ルール

検索ページでは、実在検索エンジン風に寄せすぎない。

禁止。

* Google風ロゴ
* Yahoo風ロゴ
* 白背景に巨大ロゴと検索窓だけ
* 実在検索結果に酷似した表示
* 「検索」だけの無機質なボタン

推奨。

* FICTIONAL SEARCH INTERFACE
* 記録を辿る
* アーカイブ検索
* 照合する
* 調査開始
* 記録カード
* アーカイブ照合結果

検索結果をクリックした場合は、原則として別タブで開く。

実装例の考え方。

* `<a target="_blank" rel="noopener noreferrer">` を使う
* 既存の検索ページへ戻りやすい設計にする
* スマホでもタップしやすいカードにする

## CSS修正ルール

CSS修正では、全体を壊さない。

禁止。

* 既存CSSを大幅削除する
* 全ページに同じ見た目を強制する
* `!important` を乱用する
* 原因不明のまま上書きする
* スマホ対応を消す
* ヘッダーやフッターを隠す

推奨。

* ページ単位のクラスを使う
* 影響範囲を限定する
* PC基準で整える
* media query でスマホ調整
* 余白と文字サイズを丁寧に調整する

## 最重要ルール

修正は、作品の世界を守るために行う。

work-2 では、最終的にプレイヤーが受け取るべきものは、
謎を解いた達成感だけではなく、
母から子へ残された愛情と、記録を完成させた静かな余韻である。

その体験を壊す修正は、たとえ見た目が整っていても失敗とする。

PC基準で丁寧に作り、スマホでも読めるようにし、
実在と混同させず、架空世界として自然に存在させること。
