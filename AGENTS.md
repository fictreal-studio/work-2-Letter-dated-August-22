# AGENTS.md

## 目的

このファイルは、本リポジトリで作業するAIエージェント、Codex、開発者、デザイナーに対する共通指示書である。

本リポジトリは、ARG / オンライン調査型ヒューマンドラマ作品 **work-2** を制作するためのものとする。
単なるWebサイト制作ではなく、複数の架空Webページ・記事・ログ・日記・アーカイブを通じて、プレイヤーが物語を調査し、最後にアーカイブを完成させる体験を作る。

## 作業前に必ず確認する資料

作業を始める前に、以下の資料を参照すること。

1. `work-2-設定.txt`
2. `ARG作品制作における不変原則集`
3. 既存の `AGENTS.md`
4. 既存の `DESIN.md`
5. 既存の `FIX.md`
6. 変更対象ページの HTML / CSS / JS
7. 変更対象ページとリンクしている前後ページ

資料に矛盾がある場合の優先順位は以下とする。

1. ユーザーからの最新指示
2. `work-2-設定.txt`
3. `ARG作品制作における不変原則集`
4. `AGENTS.md`
5. `DESIN.md`
6. `FIX.md`
7. 既存コードの挙動

ただし、混同リスク・著作権リスク・実在機関誤認リスクに関わる内容は、常に不変原則を優先する。

## ページ制作の基本方針

全ページに存在理由を持たせる。
雰囲気だけのページ、意味のないお知らせ、物語に関係しない装飾は作らない。

各ページは、最低限以下を満たすこと。

* そのページが物語上なぜ存在するのか明確である
* プレイヤーがそのページで何を知るのか明確である
* 次にどこへ向かうのか分かる
* 固有名詞・日付・文体が設定資料と一致している
* デザインがページの役割と合っている
* フィクション表記が適切に入っている
* PC表示を基準として完成度を担保し、スマホでも破綻しない

---

## PC基準・レスポンシブ方針

本作品のデザインは **PC表示を基準** とする。

理由は、プレイヤーが複数ページを調査し、記事・ログ・日記・アーカイブを読み比べる体験が中心であるため。
PCで見たときに、余白、情報密度、視線誘導、記事感、資料感が最も美しく成立するように設計する。

ただし、スマホ対応を捨ててはいけない。
スマホでは、PC用レイアウトを無理に縮小するのではなく、縦積み・折りたたみ・余白調整で自然に読める形にする。

必須条件。

* PC表示を最優先でデザインする
* スマホでは横スクロールを発生させない
* 文字が小さすぎない
* 主要ボタンを押しやすくする
* 画像と本文の順番が崩れない
* 重要導線が消えない
* PCとスマホで情報量が変わりすぎない

---

## フィクション表記のルール

全ページに、控えめなフィクション表記を入れる。
これは削除してはいけない。

基本表記例。

FICTIONAL STORY INTERFACE
このページは創作作品内の架空Webページです。

ページの種類に応じて、以下のように変えることはできる。

* 検索ページ：FICTIONAL SEARCH INTERFACE
* ニュース・記事ページ：FICTIONAL NEWS ARCHIVE
* アーカイブページ：FICTIONAL ARCHIVE INTERFACE
* 医療・記録風ページ：FICTIONAL RECORD INTERFACE
* メール風ページ：FICTIONAL MESSAGE INTERFACE

ただし、表記そのものを消してはいけない。
ヘッダーの存在を邪魔だからという理由で削除してはいけない。

---

## フッターの扱い

共通フッターを機械的に全ページへ流用しない。
各ページの世界観、CSS、トーンに合わせて、ページ固有の fictional footer を作る。

フッターには、必要に応じて以下を小さく入れる。

* このページは創作作品内の架空Webページです
* 実在の人物・団体・施設とは関係ありません
* 本作品にはAI生成画像または加工画像を含む場合があります
* 作中のフォームや記録は演出です

ヘッダーは変更禁止。
フッターは各ページのデザインに合わせて整える。

## 混同リスク回避

以下は禁止。

* 実在する学校名、病院名、警察署名、新聞社名の使用
* 実在する住所、電話番号、メールアドレスの使用
* 実在機関のロゴ、紋章、マークに似たデザイン
* Google、Yahoo、Gmail、LINE、X、Instagram、実在新聞社、実在病院、行政サイトに酷似したUI
* 実在ページのHTML/CSSコピー
* 実在事件を想起させる見出しや構図
* 実在人物に似すぎたAI画像
* 現実の通報・医療・行政行動に繋がる導線

使用してよいもの。

* 完全架空の地名
* 完全架空の団体名
* 完全架空の住所風表記
* 完全架空の電話番号風表記
* 完全架空のメールアドレス風表記
* 架空UIとして再構成された画面

参考デザインは使ってよい。

コピーしてはいけないもの。

* 写真
* HTML
* CSS
* ブランド体験そのもの

デザインは「それっぽい偽物」ではなく、「作品内に実在する架空UI」として作る。

## 検索ページのルール

検索ページは実在検索エンジンの模倣にしない。
Google風、Yahoo風にしない。

使用する表記。

FICTIONAL SEARCH INTERFACE

検索ボタンの文言は、作品世界に合わせる。

例。

* 記録を辿る
* 照合する
* アーカイブ検索

検索結果は、通常の検索結果ではなく、記録カード、照合結果、アーカイブ断片として見せる。

## 最重要原則

本作品の品質は、派手な演出ではなく、以下で決まる。

* 固有名詞がぶれないこと
* 時系列がぶれないこと
* 文体が人物ごとに違うこと
* ページごとの役割が明確であること
* PCで読んだときに美しいこと
* スマホでも破綻しないこと
* 本物と誤認させないこと
* しかし、作品世界として自然に存在していること
* 最後に、母からの愛情が残ること

この原則から外れる実装は、完成ではなく修正対象とする。
