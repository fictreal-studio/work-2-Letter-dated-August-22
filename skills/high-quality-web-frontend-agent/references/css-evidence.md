# CSS And Design Evidence Policy

Treat references as evidence for solving design problems, never as production source code or identities to reproduce.

## Required Record
- Provenance: local path or approved URL, capture date, checksum where local, owner, rights, and permitted use.
- Observation: measurable declarations, layout behavior, typography, states, image treatment, or media conditions.
- Interpretation: the problem solved and the visual, usability, or business effect.
- Adaptation: how the concept changes for this owner, content, mode, and implementation.
- Limits: content length, viewport, writing mode, browser support, performance, motion, and accessibility assumptions.
- Decision: approve, project-only, needs-evidence, or reject, with confidence and adoption reason.

## Prohibited Reuse
Do not copy selector trees, source bundles, branding, proprietary fonts, logos, distinctive compositions, brand-specific color systems, spacing signatures, photography treatments, or decorative motifs wholesale.

## Promotion
Inventory output remains `candidate`. Catalog approval is metadata-only. Shared CSS requires successful implementation and evaluator evidence in more than one project, except universal accessibility corrections. Browser verification and documented departures are mandatory before adoption.
