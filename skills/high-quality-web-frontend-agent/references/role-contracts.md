# Role Contracts

## Orchestrator
Own the production record, mode selection, assignments, handoffs, and exit gates. Reject undocumented assumptions and preserve evaluator independence.

## Reference / CSS Curator
Record source, capture date, checksum where applicable, rights, permitted use, and candidate inventories for reference sites, CSS, imagery, and supplied materials. Never approve or merge extracted CSS.

## Visual Art Director
Define the owner-specific thesis, three to five principles, role tokens, Japanese typography, grid, spacing, image direction, motion policy, responsive composition, and acceptance criteria. Select only analyzed reference patterns and document adaptations.

## Frontend Implementer
Build semantic, accessible, responsive code; isolate demo logic and future adapters; use dedicated assets; preserve approved Japanese phrase groups; record compromises and browser-test evidence.

## Quality Evaluator
Review rendered output before editing. Score safety/correctness and Design Core separately, using 3/5 for clean but generic work and reserving 4-5 for specific, authored, well-resolved design. Record `REV-###` findings with severity, evidence, expected result, viewport or state, owner, and status.

## Revision Engineer
Fix P0, P1, then P2 by root cause. Record affected files and states, test adjacent widths, and return work for independent verification.

## Mode Review Lenses
- **Story / Canon Reviewer (ARG):** verify canon, chronology, character voice, discovery level, player path, public plausibility, and spoiler leakage.
- **Business / Conversion Reviewer (Corporate):** verify business clarity, trust, differentiation, inquiry and recruitment paths, CTA priority, and basic SEO presentation.
